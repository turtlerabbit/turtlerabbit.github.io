    Dynamic Data - Consuming Observable Change Sets

    January 10, 2021

    이제 observable cache or observable list를 만들 수 있다.
    다음은 수행 할 수있는 다양한 범위를 설명하는 몇 가지 빠른 실행 예제다.

    Create a Derived List or Cache

    이 예에서는 observable change set에서 파생 컬렉션을 만드는 방법을 보여준다.

    1// Given a SourceList
    2var myList = new SourceList<People>();
    3
    4var oldPeople = myList
    5      .Connect()
    6      .Filter(person => person.Age > 65)  // 필터에 적용된 항목만을 포함하는
    7      .AsObservableList();  // 새로운 observable list를 만든다.

    이 패턴은 기존 컬렉션을 수정 한 다음 수정 된 컬렉션을 소비자에게 노출하려는 경우 매우 유용하다.

    The same pattern can be used with SourceCache by using .AsObservableCache() to create derived caches.

    As an alternative to .Bind(out collection) you can use .BindToObservableList(out observableList) for both SourceList & SourceCache.
    컬렉션이 새로 고침 알림을 지원하지 않기 때문에 .AutoRefresh()를 사용하는 소스에서 파생 된 read-only lists를 가져 오는 데 유용하다.

    Observing Properties of Objects in a Collection

    컬렉션이 INotifyPropertyChanged를 구현하는 개체로 구성된 경우 다음 연산자를 사용할 수 있다.

    1// WhenValueChanged 연산자는 
    2// 지정된 속성 값이 변경된 경우 관찰 가능한 값을 반환한다.
    3
    4var ageChanged = peopleDataSource
    5      .Connect()
    6      .WhenValueChanged(p => p.Age)
    7
    8#
    9
    10// WhenPropertyChanged 연산자는 
    11// 지정된 속성이 변경된 경우 부모 개체뿐 아니라 
    12// 지정된 속성의 값으로 구성된 관찰 가능 항목을 반환한다.
    13
    14var ageChanged = peopleDataSource
    15      .Connect()
    16      .WhenPropertyChanged(p => p.Age)
    17
    18#
    19
    20// WhenAnyPropertyChanged 연산자는 
    21// 속성이 변경된 경우 개체의 관찰 가능 개체를 반환한다.
    22
    23var personChanged = peopleDataSource
    24      .Connect()
    25      .WhenAnyPropertyChanged()

    Observing item changes

    바인딩은 동적 데이터의 매우 작은 부분이다.
    위의 알림 속성 변경 오버로드는 바인딩시의 예일뿐이다.
    자식 Observable이 있는 도메인 개체가 있는 경우 컬렉션 변경에 따라 항목을 구독 및 구독 취소하는 MergeMany()를 사용할 수 있다.

    1// MergeMany 연산자
    2
    3var myoperation = somedynamicdatasource
    4      .Connect()
    5      .MergeMany(trade => trade.SomeObservable());
    6        // 컬렉션이 변경되면 SomeObservable이 연결 및 연결 해제된다.

    Transformation

    1// The Transform operator allows you to map objects from 
    2// the observable change set to another object
    3
    4var myPeople = new SourceList<People>();
    5var myPeopleObservable = myPeople.Connect();
    6
    7var myTransformedObservable = 
    8        myPeopleObservable.Transform(person => new PersonProxy(person));
    9
    10#
    11
    12// The TransformToTree operator allows you to create a fully formed reactive tree 
    13// (only available for observable cache)
    14
    15var myPeople = new SourceCache<Person, string>(p => p.Name);
    16var myTransformedObservable = myPeople
    17        .Connect()
    18        .TransformToTree(person => person.BossId);

    Aggregation

    1// The Count, Max, Min, Avg, and StdDev operators allow you 
    2// to perform aggregate functions on observable change sets
    3
    4var myPeople = new SourceList<People>();
    5var myPeopleObservable = myPeople.Connect();
    6
    7var countObservable =    myPeopleObservable.Count();
    8var maxObservable =      myPeopleObservable.Max(p => p.Age);
    9var minObservable =      myPeopleObservable.Min(p => p.Age);
    10var stdDevObservable =   myPeopleObservable.StdDev(p => p.Age);
    11var avgObservable =      myPeopleObservable.Avg(p => p.Age);

    Logical Operators

    1// The And, Or, Xor and Except operators allow you 
    2// to perform logical operations on observable change sets
    3
    4var peopleA = new SourceCache<Person,string>(p => p.Name);
    5var peopleB = new SourceCache<Person,string>(p => p.Name);
    6
    7var observableA = peopleA.Connect();
    8var observableB = peopleB.Connect();
    9
    10var inBoth = observableA.And(observableB);
    11var inEither= observableA.Or(observableB);
    12var inOnlyOne= observableA.Xor(observableB);
    13var inAandNotinB = observableA.Except(observableB);
    14
    15#
    16
    17// A recent and very powerful feature is dynamic logical operators.
    18
    19var list1 = new SourceList<int>();
    20var list2 = new SourceList<int>();
    21var list3  = new SourceList<int>();
    22    
    23var combined = new SourceList<ISourceList<int>>();
    24
    25//child lists can be added or removed any time
    26combined.Add(list1);
    27combined.Add(list2);
    28combined.Add(list3);
    29
    30//The operators look after themselves 
    31var inAll = combined.And();
    32var inAny = combined.Or();
    33var inOnlyOne= combined.Xor();
    34var inFirstAndNotAnyOther = combined.Except();

    Disposal

    1// The DisposeMany operator는 
    2// observable stream에서 제거 될 때 개체가 삭제되도록 한다.
